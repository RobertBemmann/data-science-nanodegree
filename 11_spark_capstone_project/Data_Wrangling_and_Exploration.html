The columns firstName, gender, lastName, location, registration and userAgent have the same count, so let's look at these rows first. We can see that the userId contains an empty string for these rows. Therefore, we can assume that these rows belong to users that are not logged in (column auth shows 'Logged out' as well). The distinct page values for these events support our hypothesis. We can drop these rows as our goal is to predict user churn behaviour and we can not link these pagevies / events to userIds.

Define Churn¶

Once you've done some preliminary analysis, create a column Churn to use as the label for your model. I suggest using the Cancellation Confirmation events to define your churn, which happen for both paid and free users. As a bonus task, you can also look into the Downgrade events.

It turned out that there might be 2 possible definitions of churn. One could be the downgrade from a paid subscription, the other one could be the actual cancelation of the account. The second one is actually worse because a user that canceled his account might never come back. As proposed we will take the Cancellation Confirmation page as label for our model. However, keeping the count of Downgrade and Submit Downgrade pages might be a useful feature for our model.

Apparently, customers check the Downgrade page more often than they eventually submit the downgrade. We have 52 cancellations in total and 63 downgrade submission. We could also observe that customers had Downgrade pageviews and no cancellation as well as a cancellation without looking at the Downgrade page previously.

def plot_avg_page_count(page):
    plot = df_cleaned.where(df_cleaned.page == page)\
        .groupby('userId','user_canceled')\
        .count()\
        .groupby('user_canceled')\
        .agg({'count':'avg'})\
        .toPandas()\
        .plot(x='user_canceled',y='avg(count)',kind='barh',
              title='Avg. count of pageviews per group for "{}".'.format(page),
             table=True, grid=True, figsize=(8,4), rot=4);
    table = plot.tables[0]
    table.set_fontsize(10) # Setting the font size
    table.scale(1,2) # Rescaling the rows to be more readable
